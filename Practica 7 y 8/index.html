Carro a control remoto con garra bluetooth

Cristian Ulises Ríos García
Jesus Mauricio Díaz Flores
Mauro David Sánchez Arena
Ingeniería Mecatronica
Fecha: Noviembre 27, 2024
Arduino Electrónica

- Resumen -

En esta práctica, diseñamos y construimos un carro a control remoto y con una garra. Todo esto controlado por una aplicación creada por nosotros con conexión bluetooth

- Introducción -

Bluetooth es una tecnología inalámbrica de corto alcance que conecta dispositivos sin cables, como auriculares, altavoces y relojes inteligentes. Opera en la frecuencia de 2.4 GHz, consume poca energía y es ampliamente utilizada por su facilidad y compatibilidad.

- Materiales -

Los materiales utilizados para esta practica son:

Componentes Electrónicos y Eléctricos

  • 3 Servomotores grandes: para controlar los movimientos de la garra.
  • 2 Motores DC: para las ruedas traseras del coche (solo en el proyecto del carrito).
  • 1 Micro motor: para mover la dirección del coche mediante engranajes (solo en el proyecto del carrito).
  • Puente H (L298N): para controlar los motores del carrito.
  • Módulo Bluetooth: para la comunicación remota con el celular.
  • Protoboard: para facilitar las conexiones eléctricas.
  • Fuente de alimentación para protoboard: transforma la corriente de 9V a 5V.
  • Jumpers: para realizar las conexiones entre los componentes.
  • 3 Pilas de 9V:
    • Una para los servomotores.
    • Una para el Arduino.
    • Una para el puente H (solo en el carrito).

Materiales de Construcción

  • Impresiones 3D:
    • Base del coche.
    • Sistema direccional.
    • Llantas.
    • Estructura de la garra.
    • Tornillos: para ensamblar las piezas impresas y los componentes.
    • Ligas: colocadas en las llantas para mejorar el agarre, ya que eran impresas en 3D.
    • Engranajes: impresos en 3D para transmitir movimiento entre los motores y las partes mecánicas.

Herramientas y Otros:

  • Arduino Uno: como controlador principal de ambos proyectos.
  • Aplicación móvil (App Inventor): utilizada para diseñar las apps de control.
  • Cable USB: para cargar y programar el Arduino desde la computadora.

- Desarrollo Carro -

A continuación, compartimos cómo construimos nuestro carrito a control remoto con garra, utilizando diseño 3D, componentes electrónicos y programación para hacerlo funcional.

Pasos que Seguimos para Construir el Carrito:

1. Diseño y Fabricación de las Piezas Primero, diseñamos todas las piezas en 3D. Creamos la base del coche, el sistema direccional y reutilizamos unas garras que ya teníamos. Después de diseñar, imprimimos todas las piezas necesarias utilizando una impresora 3D.

2. Montaje del Sistema de Movilidad Para que nuestro coche pudiera moverse, utilizamos dos motores DC para las ruedas traseras y un micro motor para la dirección:

Motores traseros: Nos dimos cuenta de que nuestro puente H tenía espacio solo para dos motores, pero necesitábamos controlar tres. Así que puenteamos los motores DC traseros para que ambos se controlaran al mismo tiempo desde un solo canal del puente H. Micro motor para la dirección: Conectamos el micro motor al otro canal del puente H. Este motor, mediante un sistema de engranajes que diseñamos, mueve la direccional del coche.

3. Configuración de la Garra Nuestra garra tiene tres servomotores negros, que son algo pesados. Para conectarlos: Usamos una protoboard para distribuir las conexiones. Conectamos el positivo de la protoboard a una pila y el GND al Arduino. De esta forma, los servomotores se alimentan directamente de la pila pero son controlados desde el Arduino.

4. Control del Sistema con Bluetooth Finalmente, conectamos todo el sistema para controlarlo con una app: Usamos un módulo Bluetooth que conectamos al Arduino para recibir comandos desde un celular. En App Inventor, diseñamos una aplicación con varios botones. Cada botón envía una letra distinta al Arduino. Programamos el Arduino para que, al recibir una letra, ejecutara una acción específica: Mover la garra a una posición determinada. Mover las ruedas traseras hacia adelante o atrás. Girar la dirección para manejar el coche.

Creación de una Garra Robótica para Recolectar Pelotitas

En este proyecto diseñamos y construimos una garra robótica que incorporamos a un coche para recoger pelotitas. A continuación, describimos el proceso detallado de cómo la construimos y la programamos. Diseño y Construcción de la Garra La garra cuenta con tres servomotores que controlan diferentes movimientos: Movimiento vertical: El primer servomotor mueve toda la garra hacia arriba y hacia abajo. Movimiento de la muñeca: El segundo servomotor permite que la muñeca de la garra se incline en diferentes direcciones. Apertura y cierre de la pinza: El tercer servomotor está conectado a un engranaje que acciona otro engranaje, permitiendo abrir y cerrar las pinzas de la garra. Conexiones y Alimentación Conectamos los tres servomotores a una protoboard para facilitar las conexiones eléctricas. Energización de los servos: Usamos una pila de 9V para alimentar los servomotores. Puenteamos el GND de la protoboard con el GND del Arduino, asegurándonos de que ambos sistemas compartan la misma referencia. Alimentación del Arduino: Conectamos otra pila de 9V al Arduino para energizarlo. Módulo Bluetooth: También conectamos un módulo Bluetooth al Arduino, lo que nos permitió controlarlo de manera remota desde un celular. Control de la Garra con App Inventor Desarrollamos una app en App Inventor para manejar los movimientos de la garra. La app tiene cinco botones para cada servomotor, correspondientes a los ángulos 0°, 45°, 90°, 135° y 190°. Cada botón envía una letra específica al Arduino a través del módulo Bluetooth. Programación del Arduino: Al recibir una letra, el Arduino identifica qué servomotor debe moverse y lo posiciona en el ángulo correspondiente. Resultado Final Con esta configuración, logramos una garra robótica completamente funcional, capaz de recoger pelotitas con precisión. La combinación de diseño mecánico, conexiones electrónicas y control remoto nos permitió integrar la garra en un coche y ampliar sus funcionalidades, haciendo el proyecto más dinámico y útil.
